Название проекта JudgeLite

Тип проекта Web-приложение для обучения олимпиадному программированию

Жанр Образовательная платформа с автоматической проверкой решений

Целевая платформа Windows, Linux, macOS (через веб-браузер)

Язык разработки Python 3.12

Используемый движок / фреймворк Flask 2.3+, SQLAlchemy, Docker, Bootstrap 5

ЦЕЛИ ПРОЕКТА

Создание удобной платформы для изучения алгоритмов и структур данных.
Реализация системы автоматической проверки решений на Python и C++ через Docker.
Разработка личного кабинета с визуализацией прогресса (диаграммы, статистика).
Создание рейтинга пользователей на основе количества решённых задач.
Обеспечение стабильной работы при одновременной проверке нескольких решений.
ОСНОВНЫЕ ФУНКЦИОНАЛЬНЫЕ ВОЗМОЖНОСТИ

Пользовательская система Регистрация и авторизация пользователей (email, пароль, имя). Личный профиль с отображением: Общего количества решённых задач. Количества попыток. Диаграммы решённых задач по темам. Места в общем рейтинге. Рейтинг всех пользователей (топ-10 на главной странице рейтинга).

Темы и задачи Разделы (темы): введение в алгоритмы, массивы и строки, сортировка и поиск, рекурсия и ДП, структуры данных. Для каждой темы: Вкладка «Теория» (Markdown/HTML с примерами кода). Вкладка «Практика» (список задач по теме). Задачи содержат: Название, описание, примеры ввода/вывода. Ограничения по времени (1–2 секунды). JSON-набор тестов (входные данные → ожидаемый вывод). Система проверки решений Поддержка языков: Python 3.10+ и C++17. Отправка решения через текстовое поле или загрузку файла .py / .cpp. Автоматическая проверка кода в изолированных Docker-контейнерах: Компиляция C++ (один раз на все тесты). Прогон всех тестов (вход → вывод, сравнение с ожидаемым). Ограничения: время CPU, память, доступ к сети запрещён. Выдача результатов: OK (все тесты пройдены), WA (неправильный ответ), RE (ошибка выполнения), CE (ошибка компиляции), TL (превышение времени). Сохранение истории всех попыток пользователя.

Интерфейс Адаптивный дизайн на Bootstrap 5. Редактор кода с поддержкой Tab (вставка отступа). Отображение статуса задачи в списке: Решена (зелёный фон) / есть попытка, но не решена (красный фон) / не решалась (нейтральный). Хлебные крошки для навигации по темам и задачам. Пагинация (при необходимости).

API (REST) Публичное API для получения списка задач и отправки решений: GET /api/v2/tasks - список задач. GET /api/v2/tasks/ – детали задачи. POST /api/v2/submissions – отправка решения (требует авторизации).
